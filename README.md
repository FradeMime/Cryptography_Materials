## 二、Signal 协议中的核心学习点

### 1. 端到端一次一密（E2EE）

**重点协议**：

- X3DH（Extended Triple Diffie-Hellman）

- Double Ratchet（双棘轮算法）

**推荐阅读**：

- 文字讲解：https://blog.csdn.net/DoubleSandwich/article/details/126960622

- 视频讲解：https://www.bilibili.com/video/BV1Wf4y167G6/?spm_id_from=333.337.search-card.all.click

- 理解 **事后安全性（Post-Compromise Security）**

## 三、后量子密码（Post-Quantum Cryptography, PQC）学习建议

> 本部分 **目标不是数学推导**，而是：  
> **知道是什么、为什么要用、怎么在工程中用**。

### 1. 后量子密码的核心背景

**需要掌握的概念**：

- Shor 算法：
  
  - 可在量子计算机上高效破解 **RSA / ECC / DH**

- Grover 算法：
  
  - 对对称密码与哈希的影响是“平方级削弱”（密钥加倍即可）

**结论级认知**：

- RSA / ECC / DH：**不安全**（在未来量子模型下）

- AES / SHA：**仍可使用，但需更大参数**

---

### 2. 后量子密码的主要技术路线（只需认识）

| 路线    | 代表算法             | 主要用途      |
| ----- | ---------------- | --------- |
| 格密码   | Kyber, Dilithium | 密钥封装 / 签名 |
| 编码密码  | McEliece         | 加密 / KEM  |
| 多变量密码 | Rainbow（已破）      | 签名        |
| 哈希签名  | SPHINCS+         | 签名（无结构假设） |

> 当前工程界 **事实标准**：
> 
> - **Kyber（KEM）** + **Dilithium（签名）**

---

### 3. 后量子密码“怎么用”（工程视角）

**必须建立的认知**：

- 后量子密码 ≠ 量子通信

- 后量子密码 **完全运行在经典计算机上**

- 使用方式与传统密码非常相似：
  
  - KEM → 派生会话密钥 → 对称加密

**在 Signal 中的典型替换方式**：

- 用 **PQC-KEM 替换 DH**

- 或采用 **Hybrid 模式（ECC + PQC）**：
  
  - 只要有一种安全，整体就安全
